## Auth Flow

```
Desktop mode:
  1. Server generates 32-byte hex token → ~/.config/letta/ctrl-token
  2. GET / injects token into HTML: <script>window.__LETTA_CTRL_TOKEN__="..."</script>
  3. Frontend reads window.__LETTA_CTRL_TOKEN__ → _token set immediately (no prompt)
  4. Token persisted to localStorage for consistency
  5. All /api/* calls carry Bearer header; EventSource uses ?token= query param
  6. Server validates both via checkTokenValue() (timingSafeEqual)

VPS mode:
  1. LETTA_CTRL_TOKEN=disable set in systemd env
  2. checkAuth() always returns true; injected token is empty string
  3. Tailscale ACL is the auth boundary (only tailnet members can reach :8284)
```

## Data Flow: Status Polling

```
Every 5 seconds:
  Frontend pollStatus()
    → GET /api/status (with Bearer)
    → Server handleStatus()
      → systemctl show letta --user --property=ActiveState,MemoryCurrent,CPUUsageNSec,ActiveEnterTimestamp
      → systemctl show ollama         --property=...  (system-level, no --user)
      → systemctl show better-ccflare --user --property=...
      → systemctl show ccflare-docker-proxy --user --property=...
    → JSON response: { svc: { active, memory_mb, cpu_pct, uptime } }
  Frontend updates:
    → Health pills (nav bar)
    → Service cards (overview)
    → Agent strip chips
```

## Data Flow: Letta API Proxy

```
Frontend authFetch("/api/agents")
  → Server proxyLetta("/v1/agents", req)
    → fetch("http://127.0.0.1:8283/v1/agents", { headers: { Authorization: Bearer LETTA_SERVER_PASSWORD } })
    → Response passthrough (status + body + content-type)
  → Frontend receives agent list (no Letta credentials ever reach browser)
```

## Data Flow: Live Logs (pre-fetch + SSE)

```
Frontend setLogSvc("letta")
  Step 1 — instant pre-fetch (~37ms):
    → GET /api/logtail/letta?n=50 (Bearer auth)
    → Server handleLogTail("letta", 50)
      → spawnSync("journalctl", ["--user", "-u", "letta", "-n", "50", ...])
    → JSON array of 50 lines rendered immediately

  Step 2 — live SSE follow (connects in background):
    → new EventSource("/api/logs/letta?token=...")
    → Server handleLogs("letta")
      → spawn("journalctl", ["--user", "-u", "letta", "-f", "-n", "20", ...])
    → First 20 SSE events skipped (duplicates from pre-fetch)
    → Subsequent events: JSON.parse → classifyLog() → appendLogLine()
    → logLines[] buffer (max 500), filtered by logFilter input
```

## Resolved Issues (fixed 2026-03-23)

### Round 1 — Functional fixes (ADR-031)
1. **CPU metric** — Docker services use `docker stats --no-stream`. Native services use `CPUUsageNSec` delta.
2. **RAM values** — Docker services read real container memory via `docker stats`.
3. **Service card log tails** — `GET /api/logtail/:service` returns last N journal lines.
4. **SSE auth** — `checkAuth()` accepts `?token=X` query param. Frontend passes token in EventSource URL.
5. **SVC_EXTRA** — Replaced misleading static values with generic "Status: OK/—".
6. **Targeted DOM updates** — `updateSvcGrid()` does in-place updates after first render.
7. **Create agent schema** — `llm: model` → `model: model` per current Letta API.
8. **Memory block DOM IDs** — `safeId()` sanitizes labels for DOM IDs.

### Round 2 — Security hardening + stability (ADR-032)
9. **XSS: token injection** — Server uses `JSON.stringify()` to escape token in injected `<script>`.
10. **XSS: agent names** — `escHtml()` applied to all 12 agent name/model DOM insertions.
11. **XSS: block save onclick** — Replaced inline JS strings with `data-*` attributes + `this.dataset`.
12. **XSS: contenteditable** — `contenteditable="plaintext-only"` prevents HTML paste.
13. **Block label encoding** — `encodeURIComponent(label)` in frontend, `decodeURIComponent()` on server.
14. **SSE crash** — `closed` flag guards ReadableStream controller; `idleTimeout: 255` on Bun.serve.
15. **EventSource error** — `onerror` closes stream, shows "Connection lost" reconnect message.
16. **svcAction feedback** — Checks `data.ok` response, logs failures.
17. **Polling cleanup** — Recursive `setTimeout` (no stacking); `beforeunload` clears timer + EventSource.
18. **Escape key** — Closes create modal.
19. **Log filter debounce** — 150ms debounce on keystroke via `debouncedFilter()`.
20. **Long names overflow** — CSS `text-overflow:ellipsis` on `.achip-name`, `.adetname`, `.aitem-name`.
21. **LETTA_PASSWORD warning** — Server warns on startup if credentials are missing.
